## 4.1 图的基本概念 🟠

### 什么是图

- **无向图**：G = ⟨V, E⟩，E 是无序对集合（边集）。边 {u, v} 没有方向。
- **有向图**：D = ⟨V, E⟩，E 是有序对集合（弧集）。边 ⟨u, v⟩ 有方向。
- **顶点数（阶）** n = |V|，**边数** m = |E|。

### 图的分类

- **简单图**：无环、无平行边。考试中最常见。
- **多重图**：允许平行边（两个顶点间有多条边）。
- **完全图 Kₙ**：n 阶简单图，每对不同顶点间恰有一条边。
  - 无向：m = C(n, 2) = n(n-1)/2
  - 有向：m = n(n-1)

### 度与握手定理

- **无向图的度**：d(v) = 与 v 关联的边数（**环计 2**）
- **有向图的度**：出度 d⁺(v) + 入度 d⁻(v) = d(v)

**握手定理**（图论第一定理）：
- 无向图：∑ d(v) = 2m（度数之和 = 边数的两倍）
- 有向图：∑ d⁺(v) = ∑ d⁻(v) = m

> **重要推论**：任何图中**奇度顶点的个数必为偶数**。

### 度序列与可图化

- **度序列**：把图中所有顶点的度数排成一个序列。
- **可图化**：给定度序列，判断是否存在一个简单图满足该序列。

## 4.2 通路与回路 🟠

### 概念精确辨析（最容易混淆）

| 术语 | 条件 | 通俗理解 |
|------|------|----------|
| **通路 (walk)** | 顶点和边的交替序列 | "随便走"，允许重复 |
| **简单通路 (trail)** | 边不重复 | "每条边只走一次" |
| **初级通路 (path)** | **顶点不重复** | "每个顶点只走一次" |
| **回路 (closed walk)** | 起点=终点的通路 | "绕一圈回来" |
| **简单回路 (circuit)** | 边不重复的回路 | "每条边只走一次，绕回来" |
| **初级回路 (cycle)** | 顶点不重复（除起点=终点） | "圈" |

> **包含关系**：初级通路 ⊆ 简单通路 ⊆ 通路。初级最严，通路最宽。

### 重要定理

> 在 n 阶图中，若存在从 u 到 v 的通路，则必存在长度不超过 n-1 的初级通路。
>
> 在 n 阶图中，若存在过 u 的回路，则必存在长度不超过 n 的初级回路。

**直观理解**：如果通路长度 ≥ n，根据**鸽巢原理**，必有某个顶点重复出现。把重复顶点之间的那一段"裁掉"，就得到更短的通路。

## 4.3 图的连通性 🟠

### 无向图的连通性

- **连通**：两顶点间存在通路。
- **连通图**：任意两顶点连通。
- **连通分支**：极大连通子图，个数记为 p(G)。

### 割点与割边（桥）

- **割点**：删除后使连通分支数增加的点。
- **割边（桥）**：删除后使连通分支数增加的边。
- **点割集**：一组顶点，删除后使图不连通。
- **边割集**：一组边，删除后使图不连通。

### 连通度

- **点连通度 κ(G)**：使图不连通所需删除的最少顶点数。
- **边连通度 λ(G)**：使图不连通所需删除的最少边数。
- **定理**：κ(G) ≤ λ(G) ≤ δ(G)（最小度）

### 有向图的连通性（三个级别）

| 级别 | 条件 | 强度 |
|------|------|------|
| **弱连通** | 忽略方向后底图连通 | ⭐ |
| **单向连通** | 任意两点间至少单向可达 | ⭐⭐ |
| **强连通** | 任意两点间双向可达 | ⭐⭐⭐ |

> **强连通 ⇒ 单向连通 ⇒ 弱连通**，反过来不成立。

## 4.4 图的矩阵表示 🔴

### 邻接矩阵 A

- **邻接矩阵 A**：aᵢⱼ = 从 vᵢ 到 vⱼ 的边数。无向图的 A 是对称矩阵。

**邻接矩阵的幂的含义（核心考点）**：

> (Aᵏ)ᵢⱼ = 从 vᵢ 到 vⱼ **长度为 k** 的通路数目。
>
> 主对角线元素之和 = 长度为 k 的**回路总数**。

### 可达矩阵 P

- **可达矩阵 P**：pᵢⱼ=1 当且仅当 vᵢ 可达到 vⱼ。
- 求法：P = I ∨ A ∨ A² ∨ ... ∨ Aⁿ⁻¹（布尔运算，∨ 表示逻辑或）

> **为什么只到 Aⁿ⁻¹**：因为如果存在通路，一定存在长度 ≤ n-1 的通路。

## 4.5 欧拉图与哈密顿图 🔴

### 欧拉图（一笔画问题）

| 概念 | 定义 |
|------|------|
| **欧拉通路** | 经过每条边恰好一次的简单通路 |
| **欧拉回路** | 经过每条边恰好一次的简单回路 |
| **欧拉图** | 具有欧拉回路的图 |
| **半欧拉图** | 具有欧拉通路但无欧拉回路的图 |

**无向图的判定**（有充要条件）：
- 欧拉图 ⇔ 连通 + **无奇度顶点**
- 半欧拉图 ⇔ 连通 + **恰有两个奇度顶点**

**有向图的判定**：
- 欧拉图 ⇔ 强连通 + 每个顶点入度 = 出度
- 半欧拉图 ⇔ 单向连通 + 除两个顶点外入度=出度，且一个出度多1（起点），一个入度多1（终点）

### 哈密顿图（周游世界问题）

| 概念 | 定义 |
|------|------|
| **哈密顿通路** | 经过每个顶点恰好一次的初级通路 |
| **哈密顿回路** | 经过每个顶点恰好一次的初级回路 |
| **哈密顿图** | 具有哈密顿回路的图 |
| **半哈密顿图** | 具有哈密顿通路但无哈密顿回路的图 |

> **关键区别**：欧拉关心**边**，哈密顿关心**顶点**。

**哈密顿图的判定 —— 没有充要条件！**这是图论中的著名难题（NP-完全）。

**必要条件**（用于**排除**）：
- 若 G 是哈密顿图，则对 V 的任意非空真子集 S，有 p(G−S) ≤ |S|

**充分条件**（用于**确认**）：
- **Dirac 定理**：n(≥3) 阶简单图，若 δ(G) ≥ n/2，则 G 是哈密顿图。
- **Ore 定理**：n(≥3) 阶简单图，若对任意不相邻的 u,v 有 d(u)+d(v) ≥ n，则 G 是哈密顿图。

> **考试策略**：必要条件排除（证否），充分条件确认（证是）。

## 4.6 最短路径与Dijkstra算法 🔴

### 问题描述

- **带权图（网络）**：每条边带有非负权值。
- **最短路径**：两顶点间权值之和最小的通路。

### Dijkstra 算法

**思想**：贪心——每次选"当前已知最近"的顶点，更新其邻居的距离。

**算法步骤（标号法）**：
1. 初始化：源点 s 距离为 0，其余为 ∞。所有顶点标"未访问"。
2. 在未访问顶点中选当前距离最小的 u，标记为"已访问"。
3. 对 u 的每个未访问邻接点 v：若 dist[u] + w(u,v) < dist[v]，则更新 dist[v]。
4. 重复 2-3，直到所有顶点访问完毕。

> **为什么贪心正确**：因为所有边权非负，已确定最短距离的顶点永远不会再被"更优路径"更新。

### 手算 Dijkstra 的标号法

| 步骤 | 已访问 | v₁ | v₂ | v₃ | ... |
|------|--------|-----|-----|-----|-----|
| 初始 | {s} | ∞ | 3 | ∞ | ... |
| 第1步 | {s,v₂} | 5 | **3** | 7 | ... |

> **每步要做**：① 选最小距离的未访问顶点（加粗/圈出）② 用该顶点更新所有邻居 ③ 标记为已访问。

## 4.7 树的基本概念 🔴

### 什么是树

- **树**：连通且无回路的无向图。
- **森林**：每个连通分支都是树的图。
- **树叶（悬挂顶点）**：度为 1 的顶点。
- **分支点**：度 ≥ 2 的顶点。

### 树的六个等价定义（重要！）

对 n 阶 m 边的无向图 G，以下每条都**等价于**"G 是树"：

1. G 连通且无回路（原始定义）
2. G 中任意两顶点间有**唯一**初级通路
3. G 连通且 **m = n-1**
4. G 无回路且 **m = n-1**
5. G 连通且**每条边都是桥**
6. G 无回路，但在任意两不相邻顶点间加一条边，恰得到唯一初级回路

> **怎么用**：第 3 条 m = n-1 最实用——判断是不是树，先数边数。

### 重要定理

- **n 阶非平凡树至少有 2 片树叶。**

## 4.8 生成树与最小生成树 🔴

### 基本概念

- **生成树**：包含图 G 所有 n 个顶点的树子图（n-1 条边）。
- **树枝**：生成树中的边（共 n-1 条）。
- **弦**：不在生成树中的边（共 m-(n-1) 条）。
- **存在性**：**连通图必有生成树**。

### 基本回路与基本割集

- **基本回路**：每根弦 + 生成树中部分边 → 唯一初级回路（m-(n-1) 个）
- **基本割集**：每根树枝 + 部分弦 → 唯一边割集（n-1 个）

### 最小生成树（MST）

**Kruskal 算法**（避圈法）：
1. 将所有边按权从小到大排序
2. 依次取最小权边，若不与已选边构成回路则加入，否则跳过
3. 重复直到选出 n-1 条边

> **与 Prim 算法的区别**：Kruskal 对边排序，适合稀疏图；Prim 从点出发，适合稠密图。考试手算 Kruskal 更常见。

**Cayley 公式**：Kₙ 的生成树个数 = nⁿ⁻²。

## 4.9 根树与最优二叉树（Huffman） 🔴

### 根树的定义

- **根树**：有向树，一个顶点入度为 0（树根），其余入度均为 1。
- **树根**：入度 0 → "祖宗"
- **树叶**：入度=1，出度=0
- **内点**：入度=1，出度>0
- **层数 l(v)**：从树根到 v 的通路长度
- **树高 h(T)**：最大层数

### r 叉树

| 类型 | 定义 |
|------|------|
| **r 叉树** | 每个分支点至多 r 个儿子 |
| **r 叉正则树** | 每个分支点恰好 r 个儿子 |
| **r 叉完全正则树** | 正则 + 所有树叶层数等于树高 |

### Huffman 树（最优二叉树）🔴 必考

**问题**：给定 t 个权值（频率/概率），构造一棵二叉树，使带权路径长度最小。

- **树的权** W(T) = ∑ wᵢ·l(vᵢ)（每个叶子权值 × 层数）

**Huffman 算法**：
1. 作 t 片树叶，权分别为 w₁, ..., wₜ（初始时都是"树根"）
2. 在入度为 0 的顶点中，选两个权最小的，添加新分支点为它们的父亲，新分支点的权 = 两儿子权之和
3. 重复步骤 2，直到只剩 1 个入度为 0 的顶点
4. **W(T) = 所有分支点（非树叶）的权之和**

> **巧妙之处**：W(T) 不用去乘层数，直接算所有合并过程中产生的分支点权值之和就行。

## 4.10 前缀码 🔴

### 基本概念

- **前缀**：符号串开头的连续子串。
- **前缀码**：一组符号串，其中**任意两个不同串互不为前缀**。
- **二元前缀码**：只用符号 0 和 1 的前缀码。

> **为什么要前缀码**：解码时不会产生歧义。前缀码保证读到码字末尾时无需"回头看"。

### 二叉树 ↔ 前缀码的对偶关系

- 二叉树每个分支点：左标 0，右标 1
- 从树根到树叶的路径 → 一个二元码字
- 所有树叶的码字集合 → 一个二元前缀码
- 二叉**正则**树产生**唯一**的前缀码

### 最佳前缀码

- 给定字符频率，求使得**平均编码长度最短**的二元前缀码。
- **解法**：用 Huffman 算法构造最优二叉树，树叶对应的码字就是最佳前缀码！

## 本章脉络与对比总结

### 知识递进关系

```
图的基本概念（顶点/边/度/握手定理）
        ↓
通路与回路（walk/trail/path/cycle 辨析）
        ↓
连通性（无向/有向三种级别 + 割点/桥/连通度）
        ↓
矩阵表示（邻接矩阵 A → Aᵏ计数通路）
        ↓
   ┌────────────────┼────────────────┐
   ↓                ↓                ↓
欧拉图              哈密顿图          最短路径
（边不重复）        （顶点不重复）     （Dijkstra）
充要条件            无充要条件          贪心算法
   ↓
树（连通无回路，m=n-1，六个等价定义）
   ↓
生成树（Kruskal, Cayley nⁿ⁻²）
   ↓
根树 → r 叉树 → 最优二叉树（Huffman）
   ↓
前缀码（二叉树 ↔ 二元前缀码）
```

### 五个算法速查

| 算法 | 解决的问题 | 策略 | 
|------|-----------|------|
| **Fleury** | 求欧拉回路 | "不割断后路"选边 |
| **Dijkstra** | 单源最短路径 | 贪心选最近顶点 |
| **Kruskal** | 最小生成树 | 贪心选最小边（避圈） |
| **Huffman** | 最优二叉树 | 贪心合并最优两叶 |
| **Warshall** | 传递闭包 | 动态规划 |

### 三组核心辨析

**通路/简单通路/初级通路**：
- 通路：随便走 → 简单通路：边不重 → 初级通路：顶点不重（最严）

**欧拉图 vs 哈密顿图**：
| 对比项 | 欧拉图 | 哈密顿图 |
|--------|--------|----------|
| 关心谁 | 边（每条恰好一次） | 顶点（每个恰好一次） |
| 判定 | **有充要条件** | **无充要条件** |
| 考试 | 直接套判定条件 | 必要条件排除 + 充分条件确认 |

**树 vs 生成树 vs 根树**：
- 树：连通无回路的无向图
- 生成树：包含全部顶点的树子图
- 根树：有向树，有唯一"祖宗"

### 考试应对策略

1. **握手定理**：必考。记住"环计 2"，奇度顶点必偶数个
2. **通路/回路辨析**：概念判断题，画韦恩图记牢包含关系
3. **邻接矩阵幂**：填空/计算常出——Aᵏ 某位置 = 特定长度的通路数
4. **欧拉图判定**：送分题——算度数，套条件
5. **哈密顿图判定**：策略性题目——先用必要条件排除，再用充分条件确认
6. **Dijkstra 算法**：手算大题，做表一步步来
7. **Kruskal 算法**：手算大题，边排序→避圈→直到 n-1 条
8. **Huffman 算法 + 前缀码**：捆绑考，先构造最优二叉树→读编码→算平均码长
